1. 어디가 충돌 났는지 확인하기
먼저 어떤 파일이 문제인지 확인합니다.

Bash
git status
아마 both modified: "코딩 용어.txt"라고 빨간색으로 뜰 거예요. 이 파일이 범인입니다.

2. 충돌 해결하기 (파일 수정)
VScode나 메모장으로 해당 파일(코딩 용어.txt)을 열어보세요. 아래와 같은 이상한 기호들이 보일 겁니다.

Plaintext
<<<<<<< HEAD
(내 컴퓨터에 있던 내용 - 2026.03.20)
=======
(GitHub 서버에 있던 내용)
>>>>>>> origin/main
<<<<<<< 부터 ======= 사이가 내 내용입니다.

======= 부터 >>>>>>> 사이가 서버 내용입니다.

해결 방법: 저 기호들을 다 지우고, 최종적으로 남기고 싶은 내용만 남긴 뒤 저장하세요. (기호는 하나도 남기면 안 됩니다!)

3. 해결 완료 신호 보내기 (Add & Commit)
수정을 마쳤다면 Git에게 "내가 다 고쳤어!"라고 알려줘야 합니다.

Bash
# 1. 고친 파일을 다시 장바구니에 담기
git add .

# 2. 충돌 해결 완료 커밋하기
git commit -m "충돌 해결 완료"

# 3. 이제 다시 올리기
git push origin main
